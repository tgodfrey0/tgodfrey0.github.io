ROS1 Basics

A comprehensive tutorial on the basics of the Robot Operating System 1.

Brief

ROS1 (Robot Operating System 1) is an open-source framework designed to simplify the development of complex robotic systems. It provides a rich set of tools, libraries, and conventions that enable developers to create robust and scalable robot applications. This tutorial will guide you through the fundamental concepts of ROS1, its core components, and how to start developing with it.

By the end of this tutorial, you'll understand the basic architecture of ROS1, how to set up a development environment, create and build packages, and implement simple publisher-subscriber communication between nodes.

Basics

1. ROS1 Architecture

ROS1 is built on a graph-based architecture where processing takes place in nodes that may receive, post, and multiplex sensor data, control, state, planning, actuator, and other messages. The key concepts in ROS1 are:

  • Nodes: Executable programs that perform computation.
  • Topics: Named buses over which nodes exchange messages.
  • Messages: ROS data type used when subscribing or publishing to a topic.
  • Services: Request/reply interactions between nodes.
  • Master: Name service for ROS (i.e., helps nodes find each other).
  • Parameter Server: A shared, multi-variate dictionary that nodes can use to store and retrieve parameters at runtime.

2. Setting Up Your Development Environment

Before we start developing with ROS1, we need to set up our environment. We'll use ROS Noetic, which is designed for Ubuntu 20.04.

  1. Install ROS Noetic by following the official installation guide.
  2. Set up your catkin workspace:
mkdir -p ~/catkin_ws/src
cd ~/catkin_ws/
catkin_make
source devel/setup.bash

Add the source command to your ~/.bashrc file to automatically set up your environment for ROS development: 

echo "source ~/catkin_ws/devel/setup.bash" >> ~/.bashrc
source ~/.bashrc

3. Creating a ROS Package

In ROS, software is organized in packages. Let's create a simple package: 

cd ~/catkin_ws/src
catkin_create_pkg my_first_package rospy std_msgs

This creates a new package named "my_first_package" with dependencies on rospy (Python client library for ROS) and std_msgs (standard message types).

4. Writing a Simple Publisher Node

Let's create a simple publisher node that sends a "Hello, ROS!" message every second. Create a new file named `talker.py` in the `scripts` directory of your package: 

mkdir ~/catkin_ws/src/my_first_package/scripts
cd ~/catkin_ws/src/my_first_package/scripts
touch talker.py
chmod +x talker.py

Open `talker.py` in your favorite text editor and add the following code: 

#!/usr/bin/env python3

import rospy
from std_msgs.msg import String

def talker():
  pub = rospy.Publisher('chatter', String, queue_size=10)
  rospy.init_node('talker', anonymous=True)
  rate = rospy.Rate(1) # 1 Hz
  while not rospy.is_shutdown():
    hello_str = "Hello, ROS! %s" % rospy.get_time()
    rospy.loginfo(hello_str)
    pub.publish(hello_str)
    rate.sleep()

if __name__ == '__main__':
  try:
    talker()
  except rospy.ROSInterruptException:
    pass

5. Writing a Simple Subscriber Node

Now, let's create a subscriber node that listens for messages on the 'chatter' topic. Create a new file named `listener.py`: 

touch listener.py
chmod +x listener.py

Add the following code to `listener.py`:

#!/usr/bin/env python3

import rospy
from std_msgs.msg import String

def callback(data):
  rospy.loginfo(rospy.get_caller_id() + "I heard %s", data.data)

def listener():
  rospy.init_node('listener', anonymous=True)
  rospy.Subscriber("chatter", String, callback)
  rospy.spin()

if __name__ == '__main__':
  listener()

6. Building and Running Your ROS Nodes

Now that we have our nodes, let's build the package and run them:

  1. Build the package:
cd ~/catkin_ws
catkin_make
  1. In a new terminal, start the ROS master:
roscore
  1. In another terminal, run the talker node:
rosrun my_first_package talker.py
  1. In a third terminal, run the listener node:
rosrun my_first_package listener.py

You should now see the talker publishing messages and the listener receiving them!

Advanced Features

1. Launch Files

Launch files allow you to start multiple nodes at once. Create a file named `talker_listener.launch` in the `launch` directory of your package: 

mkdir ~/catkin_ws/src/my_first_package/launch
cd ~/catkin_ws/src/my_first_package/launch
touch talker_listener.launch

Add the following content to the launch file:

#!/usr/bin/env python
import rospy
import roslaunch

def launch():
  rospy.init_node('launch_node', anonymous=True)
  launch = roslaunch.scriptapi.ROSLaunch()
  launch.start()
  talker_node = roslaunch.core.Node(
    package='my_first_package',
    node_type='talker.py',
    name='talker',
    output='screen'
  )

  listener_node = roslaunch.core.Node(
    package='my_first_package',
    node_type='listener.py',
    name='listener',
    output='screen'
  ) launch.launch(talker_node)

  launch.launch(listener_node)
  rospy.spin()

if __name__ == '__main__':
  launch()

Now you can launch both nodes with a single command:

roslaunch my_first_package talker_listener.launch

2. ROS Parameters

ROS parameters allow you to set and get global variables. Let's modify our talker to use a parameter for the publishing rate: 

#!/usr/bin/env python3

import rospy
from std_msgs.msg import String

def talker():
  pub = rospy.Publisher('chatter', String, queue_size=10)
  rospy.init_node('talker', anonymous=True)
  rate = rospy.Rate(rospy.get_param('~rate', 1))  # Default to 1 Hz if not specified
  while not rospy.is_shutdown():
    hello_str = "Hello, ROS! %s" % rospy.get_time()
    rospy.loginfo(hello_str)
    pub.publish(hello_str)
    rate.sleep()

if __name__ == '__main__':
    try:
      talker()
  except rospy.ROSInterruptException:
      pass

Now you can set the rate when launching the node:

rosrun my_first_package talker.py _rate:=2

3. ROS Services

Services allow request/reply interactions between nodes. Let's create a simple service that adds two numbers: 

#!/usr/bin/env python3

import rospy
from my_first_package.srv import AddTwoInts, AddTwoIntsResponse

def handle_add_two_ints(req):
  result = req.a + req.b
  rospy.loginfo("Returning [%s + %s = %s]"%(req.a, req.b, result))
  return AddTwoIntsResponse(result)

def add_two_ints_server():
  rospy.init_node('add_two_ints_server')
  s = rospy.Service('add_two_ints', AddTwoInts, handle_add_two_ints)
  rospy.loginfo("Ready to add two ints.")
  rospy.spin()

if __name__ == "__main__":
  add_two_ints_server()

You'll need to create a service definition file and modify your package.xml and CMakeLists.txt to use custom services. Below is an example of a service definition file for the AddTwoInts service used in the previous example: 

# Request
int64 a
int64 b

---
# Response
int64 sum

To use this service, you need to update your CMakeLists.txt and package.xml files as follows: 

find_package(catkin REQUIRED COMPONENTS
rospy
std_msgs
message_generation
)

add_service_files(
  FILES
  AddTwoInts.srv
)

generate_messages(
  DEPENDENCIES
  std_msgs
)

catkin_package(
  CATKIN_DEPENDS message_runtime
)
<build_depend>message_generation</build_depend>
  <exec_depend>message_runtime</exec_depend>

After making these changes, rebuild your package then you can use the AddTwoInts service in your nodes.
